```
// Passing arguments, invoking functions
function calcSquare(param) {
  console.log("This fn will return the square of:", param);
  return param ** 2;
}

const calcPower = function (param){
  console.log("This fn will return the power of:", param.base, "to", param.exp);
  param.result = param.base ** param.exp;
};

const pows = 
{ base: 2,
 exp: 5, 
 result: undefined };
// note result is undefined when pows is initialized
  
calcPower(pows);
console.log(pows.base, "^", pows.exp, "=", pows.result, "but 5 ^ 2 =", calcSquare(5));

```

#### 1️⃣ `function calcSquare(param) { ... }`

- This is a **named function declaration**.
    
- It takes one parameter (`param`), logs it, and returns its **square** using the `**` operator (exponentiation).
    

✅ Example:

`calcSquare(5); // Logs: "This fn will return the square of: 5" // Returns: 25`

---

#### 2️⃣ `const calcPower = function (param) { ... }`

- This is a **function expression** (an anonymous function assigned to a variable).
    
- It expects an **object** (not just a number).
    
- It logs the `base` and `exp` values of that object and then **modifies** it by adding or updating a `result` property:
    
    `param.result = param.base ** param.exp;`
    

---

#### 3️⃣ `const pows = { base: 2, exp: 5, result: undefined };`

- Creates an **object** with three properties:
    
    `{   base: 2,   exp: 5,   result: undefined }`
    
- Initially, `result` is `undefined`.
    

---

#### 4️⃣ `calcPower(pows);`

- The `pows` object is passed into `calcPower()`.
    

Inside the function:

`param.result = param.base ** param.exp;`

Since `param` **refers to the same object** as `pows`, the function modifies the **original object** — not a copy!

✅ After this line, `pows` becomes:

`{ base: 2, exp: 5, result: 32 }`

---

#### 5️⃣ Final `console.log`:

`console.log(pows.base, "^", pows.exp, "=", pows.result, "but 5 ^ 2 =", calcSquare(5));`

✅ It prints:

`2 ^ 5 = 32 but 5 ^ 2 = 25`

**7.4 Return statements**: Return statements can be omitted. The data-type of the returned value is not fixed either.
```
// Return statements
const returnFn1 = function () {
  console.log("Fn1: Function without a return value");
};

const returnFn2 = function () {
  console.log("Fn2: Function with a return value");
  return 5;
};

const returnFn3 = function (jump) {
  if (jump == 2) {
    console.log("Fn3: This fn has two return types, this branch returns a number");
    return 2;
  }
  console.log("Fn3: This fn has two return types, this branch returns a string");
  return "'2'";
};

console.log("Fn1:", returnFn1(), "Fn2:", returnFn2(), "Fn3->", returnFn3(2), ",", returnFn3(3));

```
