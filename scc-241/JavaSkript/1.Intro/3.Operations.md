**4.1 Arithmetic operators**: (+, -, *, /, %, ++, --) work exactly how they work in Java/C.

**4.2 Comparison operators**: (<, >, <=, >=) also behave similar to Java/C.

**4.3 Equality Operators**: (== and ===)  
JavaScript has both loose equality (==) and strict equality (===) operators. The loose equality operator (==) performs type coercion before comparison. The strict equality operator (===) checks both value and type without coercion.
```
// 
Loose and strict equality 
5 == "5"; // true (loose equality, type coercion)
5 === "5"; // false (strict equality, no type coercion)

```

**4.4 Logical operators**: (&&, ||, !)  
In JavaScript, logical operators (&& and ||) return the value of one of the operands, not strictly a boolean true or false as in C and Java.
```
let loVar1 = "val", burton = "visor";
let cond1 = loVar1 && burton;
let cond2 = loVar1 || burton;
console.log("Values:", "loVar1&&burton:", cond1, "loVar1||burton:", cond2);
``` 
#### 2️⃣ `let cond1 = loVar1 && burton;`

The `&&` (AND) operator works like this:

- It **returns the first falsy value**, or the **last value** if all are truthy.
    

So:

- `loVar1` → `"val"` (truthy)
    
- `burton` → `"visor"` (truthy)
    
- Therefore, `loVar1 && burton` → returns `"visor"` (the **last truthy value**).
    

✅ `cond1 = "visor"`

---

#### 3️⃣ `let cond2 = loVar1 || burton;`

The `||` (OR) operator works like this:

- It **returns the first truthy value**.
    

So:

- `loVar1` → `"val"` (truthy)
    
- Since that’s already truthy, it **stops there** and returns `"val"`.
    

✅ `cond2 = "val"`



**4.5 Assignment operators**: (=, +=, -=) share similar behavior to C/Java.  

**4.6 Exponentiation Operator** : (**) The ** operator in JavaScript is used for raising a number to a power.

```
// Exponentiation operator
 let result = 2 ** 3; 
 // Results in 8 (2 raised to the power of 3) 
 let sqrt = 4 ** 0.5; // Sqrt(4) = 2
```
