So far we have explored the features of the JavaScript language.  
The examples have shown one way interaction with the HTML DOM (as an event sink). Javascript can also read, modify and update the HTML DOM nodes. The first step is to get access to a specific or set of DOM nodes. The second step is to perform the desired action on the node.

### 1Ô∏è‚É£ Select elements

`let headings = document.querySelectorAll(".heading");`

- `document.querySelectorAll(selector)` selects **all elements** in the DOM that match the CSS selector.
    
- Here, `.heading` selects **all elements with the class `heading`**.
    
- **Returns a NodeList** (like an array of DOM nodes, but not exactly an array).
    

---

### 2Ô∏è‚É£ Iterate over all selected elements

`headings.forEach(head => head.textContent = "-- " + head.textContent);`

- `forEach` loops over every element in the NodeList.
    
- `head` represents the **current heading element** in each iteration.
    
- `head.textContent` gives the **text inside the element**.
    
- The line updates the text of each heading by **prepending `--`** in front of the existing text.

![[Pasted image 20251007023639.png]]1Ô∏è‚É£ Select the element

`let remixBtnGoesPoof = document.querySelector("#remixBtn");`

- `document.querySelector(selector)` selects **the first element** that matches the CSS selector.
    
- `#remixBtn` targets an element with **id="remixBtn"**.
    
- The variable `remixBtnGoesPoof` now holds a reference to that DOM element.
    

---

### 2Ô∏è‚É£ Remove the element

`remixBtnGoesPoof.remove();`

- `.remove()` is a DOM method that **deletes the element from the page**.
    
- After this line runs:
    
    - The button (or any element with `id="remixBtn"`) disappears from the page.
        
    - It **cannot be interacted with** anymore.
        
- There‚Äôs no need to access the parent node manually; `.remove()` handles it.
    

---

### 3Ô∏è‚É£ Example

#### HTML before:

`<button id="remixBtn">Click Me!</button>`

#### JS execution:

`let remixBtnGoesPoof = document.querySelector("#remixBtn"); remixBtnGoesPoof.remove();`

#### Resulting HTML:

`<!-- button is gone -->`

![[Pasted image 20251007023848.png]]### 1Ô∏è‚É£ Create a new element

`let zombieButton = document.createElement("button");`

- `document.createElement(tagName)` creates a **new DOM element** but does **not yet add it to the page**.
    
- Here, a `<button>` element is created and stored in the variable `zombieButton`.
    

---

### 2Ô∏è‚É£ Set properties/content

`zombieButton.textContent = "Zombie";`

- `textContent` sets the **visible text inside the element**.
    
- Now the button will display:
    

`Zombie`

---

### 3Ô∏è‚É£ Add it to the DOM

`document.querySelector(".footer").appendChild(zombieButton);`

- `document.querySelector(".footer")` selects the **first element with class="footer"**.
    
- `.appendChild(zombieButton)` **adds the button as a child** of that footer element.
    
- The button is now **visible and interactable** on the page.
    

---

### 4Ô∏è‚É£ Example

#### HTML before:

`<div class="footer"></div>`

#### JS execution:

`let zombieButton = document.createElement("button"); zombieButton.textContent = "Zombie"; document.querySelector(".footer").appendChild(zombieButton);`

#### HTML after:

`<div class="footer">   <button>Zombie</button> </div>`

---

### üîë Key Points

1. `createElement` ‚Üí creates **element in memory** (not visible yet).
    
2. `textContent` ‚Üí sets the **inner text**.
    
3. `appendChild` ‚Üí **attaches the element** to the DOM so it becomes visible.
    
4. You can also set other properties, like:
    

`zombieButton.id = "zombieBtn"; zombieButton.className = "btn btn-danger"; zombieButton.addEventListener("click", () => alert("Zombie clicked!"));`

### 1Ô∏è‚É£ Select elements

`let allHeadings = document.querySelectorAll(".heading");`

- `querySelectorAll(".heading")` selects **all elements** with class `heading`.
    
- Returns a **NodeList** of matching elements.
    

---

### 2Ô∏è‚É£ Get the last element

`let lastHeading = allHeadings[allHeadings.length-1];`

- `allHeadings.length - 1` gives the **index of the last element**.
    
- `lastHeading` now references the **last `.heading` element** in the NodeList.
    

---

### 3Ô∏è‚É£ Apply style

`lastHeading.style = "background-color:red";`

- `.style` allows you to modify **inline CSS styles** of an element.
    
- Here, the **background color** of the last heading is set to red.
    
- Effect: the last heading‚Äôs background turns **red immediately on the page**.
    

---

### ‚ö†Ô∏è Important Note

While setting `style` as a string works, it **overwrites any existing inline styles**. A safer alternative is to set individual properties:

`lastHeading.style.backgroundColor = "red";`

- This only changes `background-color` without affecting other styles.
    

---

### 4Ô∏è‚É£ Example

#### HTML before:

`<h2 class="heading">Title 1</h2> <h2 class="heading">Title 2</h2> <h2 class="heading">Title 3</h2>`

#### JS execution:

`let allHeadings = document.querySelectorAll(".heading"); let lastHeading = allHeadings[allHeadings.length-1]; lastHeading.style.backgroundColor = "red";`
- `Title 3` now has a **red background**.
    
- `Title 1` and `Title 2` remain unchanged.
