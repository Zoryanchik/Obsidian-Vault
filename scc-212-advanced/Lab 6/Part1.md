## What is `foldl`?
![[Pasted image 20260218222956.png]]
`foldl` walks through a list from left to right, and keeps a **running total** (called the accumulator, `acc`). It starts with an initial value and updates it for each element.

Think of it like a snowball rolling down a hill — it starts small and picks up more snow as it goes.

---

## The structure of foldl

haskell

```haskell
foldl (\acc x -> ...) startingValue list
```

- `acc` — the running total so far
- `x` — the current element being looked at
- `startingValue` — what acc begins as
- `list` — the list to walk through

---

## Tracing through an example

Let's use `sumEvenSquares [1, 2, 3, 4]`

The starting value of `acc` is **0**.

|Step|x|Is x even?|acc before|acc after|
|---|---|---|---|---|
|1|1|No|0|0|
|2|2|Yes|0|0 + 2² = **4**|
|3|3|No|4|4|
|4|4|Yes|4|4 + 4² = **20**|

Final answer: **20** ✅

---

## Breaking down the lambda

haskell

```haskell
\acc x -> if even x then acc + x^2 else acc
```

- `\acc x ->` just means "a function that takes acc and x"
- `if even x` — check if the current number is even
- `then acc + x^2` — if yes, add its square to the running total
- `else acc` — if no, leave the running total unchanged
